# Ecosystem Current Strategy Checkpoint

Status: CURRENT STRATEGY CHECKPOINT
Authority: durable strategy record; mutable priorities require fresh review
Recorded scope: ECO-01

## Ecosystem identity

The ecosystem is the **Human-Governed AI Work System**.

Its principle is:

> One ecosystem, multiple bounded products and tools.

Its shared philosophy is:

> Controlled capability through explicit boundaries and evidence.

This checkpoint describes ecosystem-level strategy. It does not replace fresh Git, GitHub, environment, tool, or authentication verification, and it does not authorize implementation or remote mutation.

## Formal components and roles

1. **Semi-Automated AI Engineering Workflow**
   - Role: Development Governance & Execution Infrastructure.
   - Priority: first-priority mainline; primary Agentic quota allocation.
   - Scope: development governance, bounded execution, evidence, recovery, and review infrastructure for Reverb, Local AI Workbench, Gateway/n8n, and portfolio work.
   - Boundary: it is not merely a Local AI Workbench feature and does not automatically authorize runtime expansion, startup, services, MCP, or remote mutation.

2. **Reverb SDK**
   - Role: Deterministic Input & Guardrail Foundation.
   - Strategy: maintain a bounded, reusable, portfolio-grade library/checkpoint rather than unbounded expansion.

3. **Local AI Workbench**
   - Role: Local-First Technical Document & Engineering Evidence Workbench.
   - Priority: lower-priority bounded product component within the ecosystem.
   - Boundary: historical investment does not automatically justify highest priority; renewed investment requires product-value evidence.

4. **Human Approval Automation Gateway / n8n**
   - Role: External Automation & Human Approval Gateway.
   - Gateway: authority, policy, approval, bounded execution, and audit boundary.
   - n8n: future orchestration/integration surface, not the V1 authority source.

5. **Portfolio / Brand Website**
   - Role: presentation, evidence, case-study, and job-search layer.

## Dual-track strategy

### Workflow Mainline

- First priority across the ecosystem.
- Agentic quota is preferentially reserved for unresolved implementation, root-cause repair, final semantic review, and high-risk acceptance.
- Each proposed next node still requires fresh evidence and an explicit acceptance goal; first priority does not mean automatic activation.

### Ecosystem Parallel Line

- Reverb bounded checkpoint.
- Gateway/n8n specification or prototype when prioritized.
- Local AI Workbench product review and bounded validation.
- Portfolio evidence packaging from verified checkpoints.
- Resume and interview materials after evidence is sufficiently mature.

The following distinction is mandatory:

> **repo-local product mainline != ecosystem-global strategic priority**

The Local AI Workbench repository may have a product-facing mainline for that repository while the Workflow remains the ecosystem's first-priority mainline. Workbench Phase 5.4 is not an upper-level gate deciding whether the Workflow continues.

## Workflow separation direction

The Workflow/Bridge is intended to become reusable cross-project development infrastructure. `local-ai-workbench` is the first validated host and reference host, not the permanent repository boundary.

The separation design exists. Physical extraction, a new repository, migration, package publishing, import rewiring, runtime-boundary changes, and cutover remain unauthorized until a separate migration implementation node is approved and its portability criteria are met.

## Resource and reality constraints

- The user is not a senior engineer; records and workflows must reduce reviewer burden and cognitive debt.
- Codex/Work/Agentic quota and time are limited; stronger models do not expand authority.
- Portfolio and job-search value are real objectives, but claims must be supported by accepted evidence.
- Do not continue a product or tool because of sunk cost, model capability, or architectural elegance alone.
- Durable records should preserve limitations and unknowns rather than convert hypotheses into facts.

## Current strategic ordering

This is a current checkpoint, not an immutable roadmap:

1. Workflow Mainline first priority.
2. Portfolio evidence packaging should consume verified checkpoints early.
3. Reverb bounded checkpoint.
4. Gateway/n8n specification or prototype when prioritized.
5. Local AI Workbench bounded follow-up only; no broad expansion without value evidence.

Fresh evidence may update this ordering through explicit strategy review.

## Current non-activation boundaries

This checkpoint does not activate Phase 5.4, RV2-04, repository separation, a new repository, Gateway/n8n implementation, or any other implementation node. No commit, push, PR, Issue mutation, dependency change, authentication change, or runtime operation is authorized by this record.

## ECO-01 outcome

The ecosystem strategy is re-aligned as a dual-track system: Workflow Mainline first, with bounded parallel ecosystem components. Local AI Workbench validation remains valuable evidence for its component but does not replace the ecosystem strategy.
